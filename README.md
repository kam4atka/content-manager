# Расширение для Google Chrome "Content manager"

Помогает протестировать верстку сайта на переполнение.

---

### Сборка проекта

При изменении исходного кода, расположенных в директории 'source', необходимо в директории проекта выполнить следующие команды:

```shell
$ npm install
$ npm run build
```

### Установка

#### 1. Скачайте и распакуйте код расширения

Для запуска расширения вам необходима директория 'build'. Вы можете скачать и разместить ее в любое удобное для вас место.

#### 2. Откройте браузер Google Chrome

#### 3. Откройте вкладку "Расширения"

В выпадающем меню "Настройка и управление Google Chrome", найдите пункт "Дополнительные инструменты - Расширения". Либо откройте страницу по адресу [chrome://extensions/](chrome://extensions/) 

#### 4. Включите "Режим разработчика"

Переключатель находится в правом верхнем углу страницы.

#### 5. Установите расширение

Нажмите на кн. "Загрузить распакованное расширение" и укажите директорию, куда вы сохранили файлы расширения.

### Использование

После установки расширения, в правом верхнем углу панели управления, появится иконка расширения. Нажав на нее вы увидите модальное окно, которое содержит две кнопки:
- "Включить режим тестирования"
- "Выключить режим тестирования".

#### 1. Элементы управления общего характера

После включения режима тестирования на странице появится блок управления контентом страницы. Он включает в себя 4 кнопки:

- "x2", запускает сценарий, при котором скрипт проходит все элементы страницы и при условии, что в них содержиться текстовое содержимое, увеличивает данное содержимое вдвое. Позволяет протестировать как поведет себя верстка при увеличении контента.

- "lg", запускает сценарий, при котором скрипт проходит все элементы страницы и при условии, что в них содержиться текстовое содержимое, заменяет его на длинную последовательность символов без пробелов.

- "im", запускает сценарий, при котором скрипт проходит все элементы страницы и при условии, если элемент является изображением, заменяет его url на несуществующий. Позволяет протестировать поведение верстки в случае ошибки загрузки изображений.

- "bg", запускает сценарий, при котором скрипт проходит все элементы страницы и удаляет все фоновые избражения. Позволяет проверить указан ли фоновый цвет для блоков с фоновыми изображениями.

#### 2. Элементы управления частного характера

Также, после включения режима тестирования, становиться возможно тестировать блоки на переполнение. Для этого на элемент BODY добавляется событие, которое отслеживает событие 'click' на странице сайта. Таким образом, когда на элементе страницы происходит событие 'click', появляется модальное окно.

Данное модальное окно содержит следующую информацию:
- название выбранного тега элемента
- название класса(-ов) выбранного элемента
- количество элементов-потомков.

В дополнении к информации о выбранном элементе, модально окно содержит элементы управления, которые включают в себя 4 кнопки (если у элемента нет элементов-потомков, то 3 кнопки):

- "На уровень выше", позволяет получить информацию о элементе-родителе, и возможность управлять данным элементов в рамках возможностей модального окна

- "2x", при наличии элементов-потомков, позволяет увеличить их вдвое

- "+1", позволяет добавить выбранный элемент к элементу-родителю после всех имеющихся

- "-1", позволяет удалить выбранный элемент из DOM-дерева.

Модальное окно можно закрыть с помщью клавиши "ESC".

Выйти из режима тестирования можно с помощью кн. "Выключить режим тестирования".
